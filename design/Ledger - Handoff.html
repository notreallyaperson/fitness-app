Ledger — Handoff

Design tokens and component mapping for a Next.js + Tailwind v4 + shadcn/ui codebase. Warm-paper neutrals, graphite ink, one acid accent (Volt) used only for live/record states. Numbers are the hero — every metric uses tabular, slashed-zero figures.

01 Tailwind v4 @theme

Drop into app/globals.css. Variables are declared on :root / .dark and exposed to utilities via @theme inline (so bg-card, text-muted-foreground, etc. follow the active mode). Names are shadcn-native; Ledger-specific extras (elevated, success, border-strong, the metric utility) are additive.

Mapping gotcha: the brand color Volt maps to shadcn --primary, not --accent. In shadcn, --accent is the subtle hover/selected surface — here it points at the muted inset. Don't paint buttons with it.
app/globals.css

Dark-mode shadows: shadows are nearly invisible on the dark surface by design — depth there comes from border + elevated. If you want them, override --shadow-soft inside .dark with a black-based rgba. Toggle the theme by adding/removing .dark on <html> (e.g. next-themes).

03 shadcn/ui mapping

Each Ledger component maps to a stock shadcn primitive with these overrides. After npx shadcn@latest init (base color: Neutral), replace the generated tokens with §01.

Buttons→Button

Primary CTA = variant="default" → bg-primary text-primary-foreground, full-width h-[52px] rounded-lg shadow-cta. Secondary = variant="outline" → bg-card border-border-strong. Ghost & Destructive map 1:1. Icon button = size="icon" at 44×44 rounded-md. Add globally: active:scale-[0.97] transition-transform duration-[120ms] ease-[--ease-tap]. Override the default radius (shadcn ships rounded-md; our buttons are md=8px, the bottom CTA is lg=14px).

Card→Card

Default → bg-card border border-border rounded-lg shadow-soft. The focal/active card (in-progress, sheets) swaps to bg-elevated. Drop shadcn's heavier default shadow; depth is one soft level + hairlines only.

Inputs · Stepper · Switch→Input Switch

Text field = Input → bg-muted border-input rounded-md h-12 focus-visible:ring-2 ring-ring. Number stepper (weight/reps) is composed, not stock: two Button size="icon" flanking a .metric value. Switch → checked state bg-primary. RPE selector = ToggleGroup type="single" (deselectable → tap-again clears), selected item bg-primary text-primary-foreground.

Bottom sheet→Drawer

shadcn Drawer (vaul) for Add-exercise & Log-set. Style DrawerContent → bg-elevated rounded-t-xl (22px) with the grabber handle; balanced close · title · Done header. Slide uses --ease-sheet. Use Dialog only for confirm-destructive ("End session?").

Segmented · Tabs→Tabs

In-page segmented controls (units lb/kg) = Tabs with a pill TabsList bg-muted and TabsTrigger active → bg-card shadow-soft. Note: the bottom nav is NOT shadcn Tabs — it's a custom fixed bar (icon + 10px label, active text-primary, center bg-primary action). Build it by hand.

Tags & badges→Badge

Muscle/equipment tags = variant="secondary" → bg-muted text-secondary-foreground rounded-sm. PB = variant="default" → bg-primary text-primary-foreground. Streak = add a success variant → bg-success/15 text-success. Keep tags monochrome; only achievements take color.

Not in shadcn — build custom: the set-logging row, the rest timer (idle/running/done), the live duration timer, and the oversized metric numerals. All lean on the .metric utility + font-mono for countdowns. These are the brand; don't substitute a generic component.
